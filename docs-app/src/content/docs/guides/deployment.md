A professional Corex setup keeps each website in its **own Azure DevOps repo** and ships Corex framework updates
separately. This guide covers the per-site repo, deployment, branch policies, and update packaging.

## Per-site repo + App Service

- Each website is its own **Azure DevOps repo** (generated by [`make:site`](/guides/client-site/)). The Corex
  framework lives in its own repo and is updated separately — client teams never edit it.
- Host on **Azure App Service** + **Azure Database for MySQL**.
- Deploy from `develop` (staging) and `main` (production), or from tags.
- **Secrets** go in App Service application settings — never a committed `.env` or `wp-config.php`. **Uploads** are
  not committed.
- **Rollback**: redeploy the previous tag / App Service deployment slot.

## Branch policies

Protect `main` and `develop`; require, before merge:

1. a **pull-request review**,
2. a **green pipeline** (build + the headless tests),
3. **`wp corex compliance:check`** — fails the PR if it touches Corex framework folders.

```bash
# In CI: fail the build if a client PR edited the Corex framework.
wp corex compliance:check --files="$(git diff --name-only origin/develop...HEAD | paste -sd,)"
```

The check passes client plugin/theme/docs/specs changes and fails (naming them) any change under a Corex framework
folder. An approved framework change uses `--allow-framework`.

## Packaging a Corex update

The framework update *mechanism* (in-admin "update available") ships already; to **produce** an update:

```bash
wp corex package:update 0.25.0 --download-url=https://example.com/corex/corex-0.25.0.zip --changelog="Bug fixes."
```

This emits the `manifest.json` the self-update mechanism reads (version / requires / requires_php / tested /
download_url / changelog) for the **framework only** — never client code, content, uploads, or secrets. Build the
ZIP from the planned framework paths and publish it + the manifest (e.g. a GitHub Release / static host).

## Read the docs locally

```bash
wp corex docs:sync    # locate the built docs to copy into .corex/docs/ (git-ignored)
wp corex docs:serve   # how to run the docs app locally (no hosted site needed)
```
