# Agent roles & the Role Gate

CoreX is built by people *and* AI agents, and it is now used to build real client sites. To keep framework work
and client work from colliding, every session runs through four gates in order:

> **Role Gate** decides *where* you work · **Spec Kit** decides *what* to build · **Guard Gate** decides
> *whether* code/docs are safe to ship · **UI/UX ProMax** decides *whether* visible UI/design is good enough.

Classify the session into exactly one mode **before editing**. Copy/paste prompts for each mode are in
[AI agent start prompts](./ai-agent-start-prompts.md).

## The four modes

### 1. CoreX Framework Mode
Working *on the framework itself*.

- **Edits:** `plugins/`, `addons/`, `packages/`, root `theme/`, root `specs/`, root `docs/`, `docs-app/`,
  `ROADMAP.md`, root `PROGRESS.md`, CoreX admin/login/docs UI, release/versioning.
- **Source of truth:** root `AGENTS.md`/`CLAUDE.md`, `specs/constitution.md`, `COREX-WORKING-GUIDE.md`,
  `COREX-FRAMEWORK.md`, root `PROGRESS.md`, `ROADMAP.md`, the active root spec.
- **Must not:** edit `sites/<client>/` unless explicitly authorized.

### 2. Client Site Mode
Working *on a generated client/company site*.

- **Edits:** `sites/<client>/` only — client plugin/theme, pages, blocks, templates, content, branding.
- **Source of truth:** the root files for global safety rules, plus `sites/<client>/AGENTS.md`/`CLAUDE.md`/
  `PROGRESS.md`/`DECISIONS.md` and `sites/<client>/specs/`.
- **Must not:** continue the CoreX framework roadmap; edit `plugins/`, `addons/`, `packages/`, root `theme/`,
  root `specs/`, `ROADMAP.md`, or root `PROGRESS.md` unless explicitly authorized. For a framework bug, **stop and
  open a CoreX Framework Mode task**.

### 3. Deployment Mode
Packaging and shipping.

- **Edits:** the `dist` builder, `azure-pipelines.yml`, deploy scripts, release artifacts, runtime-file protection,
  rollback.
- **Must not:** make client-design or framework-product changes beyond what packaging requires.

### 4. Docs/Planning Mode
Documentation and planning.

- **Edits:** docs, specs, roadmap, decisions, prompts, handoffs.
- **Must not:** ship runtime code unless explicitly authorized.

## Never edited as source

`wp/wp-content/` and `dist/` are runtime/build output. `dist/` is generated by the shared-host builder and is
git-ignored — it is never committed, and the server receives only its contents. See
[Shared-host dist](../05-deployment/shared-host-dist.md).

## Source layout

```text
# framework source (root)
plugins/  addons/  packages/  theme/  specs/  docs/  docs-app/

# client source
sites/<client>/
  <client>-site/     # client plugin (app/business code)
  <client>-theme/    # client theme (presentation)
  AGENTS.md  CLAUDE.md  README.md  PROGRESS.md  DECISIONS.md
  specs/  docs/
```

Reusable blocks/components belong in the **framework** plugins/add-ons. Company-specific blocks, templates, pages,
assets, content, and logic belong in **`sites/<client>/`**.
