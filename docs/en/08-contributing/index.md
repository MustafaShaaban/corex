# Contributing

How to make a change to Corex and get it merged. This page is the **on-ramp**: it walks a first-time
contributor through the path and links to the authoritative sources rather than duplicating their rules.

> **Status:** scaffolded; authored in **phase D9**.

## The path (planned)

1. Read [the constitution](../../../specs/constitution.md) and [`COREX-WORKING-GUIDE.md`](../../../COREX-WORKING-GUIDE.md).
2. Open a spec first (Spec Kit) — code never precedes its spec (Principle X).
3. Build test-first; run the **Guard Gate** on your diff.
4. Branch off `develop`, use Conventional Commits, open a PR; CI must be green.
5. Update `PROGRESS.md` + `DECISIONS.md`; end with a NEXT STEP block.

See also [04 · Team workflow](../04-team-workflow/) for the full workflow.
