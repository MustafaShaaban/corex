# Corex Documentation

The four canonical reference documents live at the **repository root** (they are agent entry
points per COREX-WORKING-GUIDE.md §A.2):

- [`COREX-FRAMEWORK.md`](../COREX-FRAMEWORK.md) — architecture reference.
- [`COREX-WORKING-GUIDE.md`](../COREX-WORKING-GUIDE.md) — how we work + continuity protocol.
- [`COREX-EMAIL-ADDON.md`](../COREX-EMAIL-ADDON.md) — Corex Mail add-on spec.
- [`COREX-SPECKIT-START.md`](../COREX-SPECKIT-START.md) — build sequence and module order.

This `docs/` directory holds **derived and supplementary** documentation as the framework
grows — e.g. `HOOKS.md`, `MODES.md`, and the output of `wp corex docs:generate` (hooks,
routes, CLI, abilities derived from code). It is intentionally empty until that content exists.
