# Design Handoffs

Store one focused, approved design-to-engineering handoff per implementation area here.

Each handoff should define:

- approved scope and explicit exclusions;
- screens, components, variants, and content constraints;
- interaction and state behavior;
- desktop, tablet, and mobile behavior;
- LTR, RTL, and mixed-script behavior;
- keyboard, focus, contrast, motion, and assistive-technology expectations;
- performance and conditional-asset expectations;
- design tokens and reusable primitives;
- open questions and approval evidence.

A handoff is an input to an engineering spec. It is not permission to implement code directly.
